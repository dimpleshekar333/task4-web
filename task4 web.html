Task 4 — Make website mobile-friendly
  • Objective
  • Hints
  • Outcome

Objective: Convert desktop-only page to mobile-friendly layout

Tools: your HTML file, VS Code, Chrome DevTools. Deliverable: updated CSS with responsive design working on mobile viewport.

Hints / Mini Guide

  1. Open the HTML page in an editor and identify fixed-width elements.
  2. Use media queries targeting max-width: 768px to change layout.
  3. Stack columns vertically and adjust font sizes on small screens.
  4. Make the nav collapse into a toggle (hamburger) for small viewports.
  5. Ensure images scale inside their containers: img { max-width:100%; height:auto; }.
  6. Test using Chrome DevTools device toolbar and fix overflow/scroll issues.

Example content

This paragraph shows how text reflows. On desktop you may see multi-column content — but on phones it should stack into a single column for easy reading.

Resize the browser width or open DevTools -> Toggle device toolbar to simulate smaller screens.

Checklist

  • Remove fixed-width containers
  • Use fluid widths (%, max-width)
  • Use stack layout at <=768px
  • Collapse navigation for small screens

Outcome

Grasp media queries, mobile-first design, and flexible layouts. This sidebar demonstrates a secondary column that becomes stacked on narrow screens.

Quick CSS snippet 
/* mobile-first */
.container { width:100%; max-width:1100px; }
img { max-width:100%; height:auto; }

/* responsive breakpoint */
@media (max-width:768px){
  .grid { grid-template-columns: 1fr; }
  .nav-list { display:none; }
}
